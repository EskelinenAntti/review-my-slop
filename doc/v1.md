# v1 Baseline

Dense inventory of behavior and structure that existed before the v2 development session.

## Product Behavior

- Go TUI for reviewing difftastic-rendered Git diffs.
- Startup command: `go run .`.
- Required external tools: `git`, `difft`; GitHub review features additionally require authenticated `gh`.
- Starts on local uncommitted changes from `git diff` rendered with `difft --color=always`.
- Detects active GitHub PR asynchronously with `gh pr view`.
- If PR context exists, `Tab` switches from local uncommitted changes to branch changes using `base...HEAD`.
- Non-interactive fallback: if raw terminal setup fails, print the loaded diff and a short message that interactive review needs a terminal.

## Navigation And Selection

- `j` / `k` and arrow keys move between changed/selectable diff rows.
- `g` / `G` jump to first/last changed row.
- `Ctrl-d` / `Ctrl-u`, `PageDown` / `PageUp` move by half page.
- `h` / `l` switch review target between old/left and new/right side on side-by-side rows.
- `v` starts or clears visual selection.
- Visual selection is constrained to one file and one side.
- Selection range sorts by target line before use, so reverse selections still submit ordered line ranges.
- Selected diff row is kept visible when status references it.

## Editor Workflow

- `e` / `Enter` opens selected new-side line in `$VISUAL`, then `$EDITOR`, then `vi`.
- Deleted/old-side lines are not opened; UI reports `Cannot open deleted line <file>:<line>`.
- Review bodies and review summaries open in a temporary markdown file through the same editor path.

## GitHub Review Workflow

- PR metadata detected via `gh pr view --json id,number,headRefOid,headRepository,headRepositoryOwner,baseRefOid`.
- Pending review detection via GraphQL: current viewer + pending PR reviews + draft comment count.
- `R` starts a pending GitHub review.
- `c` posts an inline PR comment immediately, or adds a draft comment if review mode is active.
- `s` opens a suggestion template for new/right-side selections only.
- Suggestions are populated from `git show <head>:<path>` or `<base>:<path>` depending on selected side.
- `p` submits active pending review as `COMMENT`.
- `D` deletes active pending review.
- GitHub REST and GraphQL payload construction lived inline in `main.go`.

## Renderer And UI

- Main diff viewport is dense, keyboard-first, no mouse workflow.
- Rendered terminal output is the source of truth for UI behavior.
- Virtual-screen tests parse ANSI output into cells for text/style assertions.
- Cursor is drawn with inverse video on selected side only.
- Diff colors outside the cursor are preserved where supported.
- Cursor suppresses diff colors inside selected inverse span.
- Help text reflects mode/PR/review availability.
- Active draft review help shows submit/delete actions and hides `R start review`.
- GitHub destructive action is explicit as `D delete draft`.
- Bottom UI used three rows: status row, message row, help row.
- Status row included selection count, selection mode, draft review summary, PR state, side marker, file, and line.
- Status row avoided dumping long selected line content.
- Startup placeholder text said `Loading uncommitted changes...`.

## Diff Parsing And Untracked Files

- Changed-line fallback parsed `git diff --no-ext-diff --unified=0`.
- Difftastic row parsing attempted to infer selectable rows from rendered diff text and ANSI colors.
- Side-by-side rows created `displaySelection` with `Left`, `Right`, active `Ref`, and `Split`.
- If difftastic parsing failed, selections fell back to refs from `git diff`.
- Untracked file refs were collected with `git ls-files --others --exclude-standard` and `os.Open`.
- Untracked files were appended to rendered diff only as an `Untracked files:` notice, not as full selectable diff content.

## Code Shape

- Most behavior lived in one large `main.go`.
- Existing internal packages:
  - `internal/ansi`: ANSI stripping, visible length, truncation, sequence end detection.
  - `internal/keys`: key decoding and key constants.
- Tests were mostly in `main_test.go` and `screen_test.go`.
- `screen_test.go` contained the virtual terminal parser and renderer snapshot/style assertions.
