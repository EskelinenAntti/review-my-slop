# V4 - Git diff first

1. Keep Review My Slop consistent with `git diff`.
    1. The executable is `slop`.
    2. The fallback terminal message tells users to run `slop`.
    3. The README documents passing normal `git diff` arguments to `slop`.
    4. The README documents both `git slop` via Git alias and shell-level `git diff` replacement.
2. Remove the `Tab` shortcut and footer hint for switching between local and branch changes.
    1. Diff selection should come from the command arguments instead of an in-app source toggle.
3. Add `o` as a PR shortcut when PR context exists.
    1. The footer hint is `o PR`.
    2. The shortcut runs `gh pr view --web`.
4. Preserve review position after editing a file.
    1. Returning from `$VISUAL` or `$EDITOR` keeps the cursor on the same file, line, and side when that changed line still exists.
    2. When reviewing local changes, returning from the editor reloads the diff so edits are reflected immediately.
